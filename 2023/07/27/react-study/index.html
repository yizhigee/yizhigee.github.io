useStae

state实际上是一个被React管理的变量
当我们通过setState()修改变量的值时，会触发组件的自动重新渲染
只有state值发生变化时，组件才会重新渲染
当state的值是一个对象时，修改时是使用新的对象去替换已有的对象
当通过setState去修改一个state时，并不表示修改当前的state，它修改的是下一次渲染的state值
setState()会触发组件的重新渲染，它是异步的

num 代表原始值。
setNum用于修改值
useState(1)，1代表默认值
prevState : 修改一个state的值而需要依赖于前边的值进行计算时prevState 获取上一次state值，通过回调函数进行修改，保证每次拿到的是最新的值

useRef

钩子函数只能在函数组件或自定义钩子
钩子函数只能在函数组件中调用

当需要一个对象不会因为组件的渲染而改变的时候可以使用useRef

useContext

将Context对象设置到一个单独的模块中并设置为默认导出

如果想访问到Context中的数据，我们需要先将Context引入到当前组件中，然后通过Context对象访问其中的数据。

方式一
通过Consumer标签来访问到Context中的数据

方式二
通过useContext()，将Context对象作为参数传递给钩子函数

App中使用

获取Context时，React会在它的外层查找最近的Provider，然后返回它的Context值。如果没有找到Provider，则会返回Context模块中设置的默认值。（就近原则）

useEffect

目前理解下来 作用类似于vue中的onMounted和onUpdated
又有点类似于vue中的watch
区别：
watch:
watch是依赖响应式对象，当依赖对象发生变化时候，，副作用函数会被触发执行
watch 第一个参数是依赖对象，第二个参数是副作用函数
useEffect:
第一个参数是副作用函数，第二个参数是一个依赖数组
当依赖数组中的变量发生变化时，副作用函数会重新执行（这一点我觉得类似于onUpdated）
如果不传依赖数组，默认副作用函数在每次组件渲染后都会执行（这一点我觉得类似于onMounted）

useReducer

Reducer主要是为了整合比较复杂的state

参数：
reducer

state所有操作都在reducer中进行，返回值是新的state值
reducer在执行的时候会收到两个参数
state 【当前最新的state】
action:【需要一个对象，对象中存储dispatch发送的指令】

initialArg

state的初始值

返回值
state

state的值

dispatch

state的修改 派发器
通过派发器可以发送操作state的指令
具体修改行为由 reducer 执行
